# 커밋 메시지 작성 필수 가이드라인

## 🚨 모든 커밋 시 반드시 준수해야 할 규칙

### 커밋 메시지 형식 (필수)

```
<type>: <description>

<body>
```

### Type 종류 (필수 선택)

- `feat:` - 새로운 기능 추가
- `fix:` - 버그 수정
- `style:` - UI/스타일 변경
- `refactor:` - 코드 리팩토링
- `test:` - 테스트 추가/수정
- `docs:` - 문서 변경
- `chore:` - 빌드, 설정 등

### Description 작성 규칙 (필수)

- **한글로 작성** (영어 파일명/기술용어 제외)
- **50자 이내**
- **첫 글자 소문자**
- **마침표 없음**
- **명령형으로 작성** ("추가한다" ❌, "추가" ✅)

### Body 작성 규칙 (권장)

```bash
feat: spotpin 컴포넌트 기본 구현

- 마커 아이콘 생성 함수 추가
- 클릭 이벤트 핸들링 구현
- 네이비 테마 적용
- Requirements 1.2, 2.1 대응
```

## 📝 커밋 메시지 예시

### ✅ 올바른 예시

```bash
feat: spot detail 페이지 구현
fix: useSpotDetail API 응답 처리 수정
style: spotpin 마커 ui/ux 개선
test: spot 데이터 직렬화 속성 테스트 추가
```

### ❌ 잘못된 예시

```bash
feat: SpotDetail 페이지 구현  # 첫 글자 대문자 ❌
Fix: 버그 수정              # 타입 대문자 ❌
feat: 기능을 추가했습니다.    # 과거형 ❌
update: 스타일 변경         # 잘못된 타입 ❌
```

## 🔄 논리적 커밋 분할 전략

### 기능별 분할 (권장)

```bash
# 1. 기본 구현
feat: spotpin 컴포넌트 기본 구현

# 2. 스타일링
style: spotpin 마커 스타일링 추가

# 3. 이벤트 처리
feat: spotpin 클릭 이벤트 핸들링 구현
```

### 커밋 크기 가이드

- **이상적**: 100-300줄
- **최대**: 500줄
- **500줄 초과**: 여러 커밋으로 분할 필수

## 📋 Requirements 참조 (필수)

모든 커밋에 관련 Requirements 번호 포함:

```bash
feat: 스팟 미리보기 팝업 구현

- 팝업 UI 컴포넌트 추가
- 필수 정보 표시 (이름, 사진, 설명, 주소)
- 상세보기 버튼 구현
- Requirements 2.2, 2.3, 2.4 대응
```

## ⚠️ 커밋 전 체크리스트

1. **타입 확인**: feat, fix, style, test, docs, chore 중 선택
2. **한글 작성**: 모든 설명은 한글로 (기술용어 제외)
3. **길이 확인**: description 50자 이내
4. **형식 확인**: 첫 글자 소문자, 마침표 없음
5. **Requirements**: 관련 요구사항 번호 포함
6. **논리적 분할**: 하나의 커밋 = 하나의 논리적 변경사항

이 가이드라인을 따르지 않으면 커밋이 거부될 수 있습니다.
