# 버그 수정 요구사항 문서

## 소개

지도 UI 관련 4가지 버그를 수정합니다:
1. 카테고리 필터 클릭 시 "Map container is being reused by another instance" 에러 발생
2. 순례갤러리 피드 아이템이 1개일 때 이미지가 과도하게 크게 표시됨
3. 코스 지도(RouteMap) 타일이 어긋나거나 깨져서 표시됨
4. 근접 핀들의 호버/클릭 이벤트 충돌로 모달/미리보기가 깨짐

이 버그들은 사용자 경험을 직접적으로 저해하며, 특히 버그 1은 에러 발생 후 재시도해야 하는 불편을 초래합니다.

## 버그 분석

### 현재 동작 (결함)

**버그 1: 지도 컨테이너 재사용 에러 (PilgrimageMap.tsx)**

1.1 WHEN 메인 페이지에서 카테고리 필터를 클릭하여 spots 배열이 변경되면 THEN MapContainer 컴포넌트가 리렌더링되면서 기존 Leaflet 인스턴스를 정리하지 않고 새로 생성하려 하여 "Map container is being reused by another instance" 에러가 발생한다

1.2 WHEN 에러 발생 후 사용자가 재시도하면 THEN 필터링된 결과가 정상 표시되지만, 첫 번째 시도에서 항상 에러가 발생하여 사용자 경험이 저하된다

**버그 2: 갤러리 피드 레이아웃 (CheckInGallery.tsx)**

1.3 WHEN 순례갤러리에서 피드 아이템이 1개만 있을 때 THEN grid-cols-2 레이아웃에서 해당 아이템이 전체 너비를 차지하여 이미지가 과도하게 크게 표시된다

1.4 WHEN 피드 아이템이 2~3개로 적을 때 THEN 그리드 아이템에 최대 너비 제한이 없어 화면 크기에 따라 비정상적으로 큰 이미지가 표시될 수 있다

**버그 3: 코스 지도 렌더링 깨짐 (RouteMap.tsx)**

1.5 WHEN 코스 지도(RouteMap)가 렌더링될 때 THEN 타일이 어긋나거나 깨져서 표시된다 (API 응답 데이터는 정상)

1.6 WHEN RouteMap 컴포넌트가 마운트된 후 컨테이너 크기가 변경되거나 초기 렌더링 시점에 컨테이너 크기가 확정되지 않은 상태에서 THEN invalidateSize() 호출 타이밍이 맞지 않아 타일 정렬이 깨진다

**버그 4: 핀 클러스터 모달 깨짐 (SpotPin.tsx)**

1.7 WHEN 여러 핀이 지도 상에서 가까이 모여있을 때 호버하면 THEN 근접 핀들의 mouseover/mouseout 이벤트가 연쇄적으로 발생하여 미리보기가 깜빡이거나 잘못된 스팟의 미리보기가 표시된다

1.8 WHEN 가까이 모여있는 핀들 중 하나를 클릭하려 할 때 THEN 이벤트 충돌로 인해 의도한 핀이 아닌 다른 핀이 선택되거나 미리보기가 정상 동작하지 않는다

### 기대 동작 (정상)

**버그 1: 지도 컨테이너 재사용 에러 수정**

2.1 WHEN 메인 페이지에서 카테고리 필터를 클릭하여 spots 배열이 변경되면 THEN MapContainer가 기존 Leaflet 인스턴스를 올바르게 재사용하거나 정리한 후 새로 생성하여 에러 없이 필터링된 결과를 즉시 표시해야 한다 (SHALL)

2.2 WHEN 카테고리 필터를 반복적으로 빠르게 전환해도 THEN 에러 없이 매번 정상적으로 필터링된 스팟 핀이 표시되어야 한다 (SHALL)

**버그 2: 갤러리 피드 레이아웃 수정**

2.3 WHEN 순례갤러리에서 피드 아이템이 1개만 있을 때 THEN 해당 아이템이 그리드 셀 하나의 크기로 제한되어 적절한 크기로 표시되어야 한다 (SHALL)

2.4 WHEN 피드 아이템이 2~3개로 적을 때 THEN 각 아이템이 최대 너비 제한 내에서 일관된 크기로 표시되어야 한다 (SHALL)

**버그 3: 코스 지도 렌더링 수정**

2.5 WHEN 코스 지도(RouteMap)가 렌더링될 때 THEN 타일이 정상적으로 정렬되어 깨짐 없이 표시되어야 한다 (SHALL)

2.6 WHEN RouteMap 컴포넌트가 마운트되거나 컨테이너 크기가 변경될 때 THEN 적절한 타이밍에 invalidateSize()가 호출되어 타일이 올바르게 렌더링되어야 한다 (SHALL)

**버그 4: 핀 클러스터 모달 수정**

2.7 WHEN 여러 핀이 지도 상에서 가까이 모여있을 때 THEN 마커 클러스터링을 적용하거나 근접 핀 간의 이벤트 충돌을 방지하여 각 핀의 미리보기가 정상적으로 표시되어야 한다 (SHALL)

2.8 WHEN 가까이 모여있는 핀들 중 하나를 클릭할 때 THEN 의도한 핀이 정확하게 선택되고 해당 스팟의 미리보기가 정상 동작해야 한다 (SHALL)

### 변경 없는 동작 (회귀 방지)

3.1 WHEN 카테고리 필터 없이 전체 스팟을 표시할 때 THEN 기존과 동일하게 모든 스팟 핀이 지도에 정상 표시되어야 한다 (SHALL CONTINUE TO)

3.2 WHEN 지도에서 줌인/줌아웃, 드래그 등 기본 인터랙션을 수행할 때 THEN 기존과 동일하게 정상 동작해야 한다 (SHALL CONTINUE TO)

3.3 WHEN 순례갤러리에서 피드 아이템이 4개 이상일 때 THEN 기존 grid-cols-2/3/4 반응형 레이아웃이 동일하게 유지되어야 한다 (SHALL CONTINUE TO)

3.4 WHEN 갤러리에서 이미지를 클릭하여 상세 모달을 열 때 THEN 기존과 동일하게 CheckInDetailModal이 정상 표시되어야 한다 (SHALL CONTINUE TO)

3.5 WHEN 코스 지도에서 Polyline, 번호 마커, 시작 지점 마커 등이 렌더링될 때 THEN 기존과 동일한 스타일과 동작으로 표시되어야 한다 (SHALL CONTINUE TO)

3.6 WHEN 코스 지도에서 스팟 마커를 클릭하여 Popup을 열 때 THEN 기존과 동일하게 스팟 정보가 표시되어야 한다 (SHALL CONTINUE TO)

3.7 WHEN 핀이 서로 충분히 떨어져 있을 때 THEN 기존과 동일하게 개별 핀의 호버/클릭 이벤트가 정상 동작해야 한다 (SHALL CONTINUE TO)

3.8 WHEN 모바일에서 핀을 터치할 때 THEN 기존과 동일하게 Bottom Sheet가 정상적으로 열려야 한다 (SHALL CONTINUE TO)

3.9 WHEN 현재 위치 버튼을 클릭할 때 THEN 기존과 동일하게 GPS 위치로 이동해야 한다 (SHALL CONTINUE TO)
