# 사용자 핵심 여정 UX 강화 요구사항

## 소개

이전 세션에서 관리자 품질 신고, 미디어 관리, 이미지 안전화, 초기 진입 안정화가 완료되었다. 이번 사양은 그 기반 위에서 일반 사용자가 핵심 여정인 `랜딩 → 지도 탐색 → 스팟 상세 → 인증/길찾기/제보 → 갤러리/작품/코스 확장`을 혼란 없이 따라갈 수 있도록 내비게이션, 복귀 경로, 갤러리 신뢰 신호, 스팟 제보 진입점, 모바일 지도 조작성을 정리한다.

## 용어 정의

- **핵심 지도 진입점**: 사용자가 전역 헤더 또는 랜딩 CTA에서 `/map`으로 이동하는 명시적 경로
- **스팟 복귀 경로**: 스팟 상세 페이지에서 지도 탐색 화면으로 돌아가는 링크 또는 버튼
- **갤러리 신뢰 신호**: 총 인증 수와 오늘 인증 수처럼 서비스 활성도를 보여주는 실제 집계 정보
- **제보 허브**: 정보 보완, 상태 신고, 품질 신고를 한 곳에서 선택하게 하는 스팟 상세 CTA 영역
- **모바일 지도 필터 패널**: 작은 화면에서 지도 면적을 보존하기 위해 검색/카테고리 필터를 열고 닫는 UI

## 요구사항

### 요구사항 1: 지도 탐색 전역 진입점 노출

**사용자 스토리:** 사용자로서, 어느 화면에서든 핵심 지도 탐색으로 즉시 이동하고 싶다.

#### 수용 기준

1. WHEN 사용자가 앱 전역 헤더를 보면, THE Header SHALL `지도 탐색` 메뉴를 표시한다
2. WHEN 사용자가 `지도 탐색` 메뉴를 클릭하면, THE Header SHALL `/map`으로 이동한다
3. WHEN 현재 경로가 `/map`이면, THE Header SHALL `지도 탐색` 메뉴를 활성 상태로 표시한다
4. WHEN 모바일 메뉴가 열리면, THE Header SHALL `지도 탐색` 항목을 모바일 메뉴에도 표시한다
5. IF 기존 작품/갤러리/코스/스팟 등록 메뉴가 존재하면, THEN THE Header SHALL 해당 메뉴를 제거하지 않는다

### 요구사항 2: 스팟 상세 복귀 경로 정확화

**사용자 스토리:** 사용자로서, 스팟 상세를 본 뒤 예측 가능하게 지도 화면으로 돌아가고 싶다.

#### 수용 기준

1. WHEN 사용자가 스팟 상세 상단의 `지도로 돌아가기`를 클릭하면, THE SpotDetailPage SHALL `/map`으로 이동한다
2. WHEN 스팟 상세 에러 화면에서 사용자가 복귀 CTA를 클릭하면, THE SpotDetailPage SHALL `/map`으로 이동한다
3. WHEN 스팟을 찾을 수 없는 화면에서 사용자가 복귀 CTA를 클릭하면, THE SpotDetailPage SHALL `/map`으로 이동한다
4. IF 루트 `/`가 방문 쿠키에 따라 `/welcome` 또는 `/map`으로 분기되더라도, THEN SpotDetailPage SHALL 복귀 링크에 `/`를 사용하지 않는다

### 요구사항 3: 갤러리 실제 통계와 실제 탭 컴포넌트 사용

**사용자 스토리:** 사용자로서, 갤러리가 실제로 활성화되어 있는지 숫자와 탭 UI를 통해 신뢰하고 싶다.

#### 수용 기준

1. WHEN 갤러리 페이지가 로드되면, THE GalleryPage SHALL `/api/checkins/stats`에서 총 인증 수와 오늘 인증 수를 조회한다
2. WHEN 통계 조회가 성공하면, THE GalleryHeader SHALL 실제 총 인증 수와 오늘 인증 수를 표시한다
3. WHILE 통계 조회가 진행 중이면, THE GalleryHeader SHALL placeholder `--` 대신 로딩 상태를 표시한다
4. IF 통계 조회가 실패하면, THEN THE GalleryHeader SHALL 실패 상태를 사용자 친화적으로 표시하고 페이지 나머지 기능은 유지한다
5. WHEN 갤러리 탭이 렌더링되면, THE GalleryPage SHALL placeholder 탭이 아닌 기존 `GalleryTabs` 컴포넌트를 사용한다

### 요구사항 4: 스팟 상세 제보 진입점 통합

**사용자 스토리:** 사용자로서, 스팟 정보에 문제가 있을 때 어떤 제보를 해야 하는지 한 곳에서 쉽게 선택하고 싶다.

#### 수용 기준

1. WHEN 스팟 상세 페이지가 렌더링되면, THE SpotDetailPage SHALL `정보 제보 및 수정` 영역을 하나만 표시한다
2. WHEN 사용자가 제보 유형을 선택하면, THE SpotDetailPage SHALL `정보 수정 제안`, `상태 신고`, `품질 신고` 중 하나를 명확히 선택하게 한다
3. WHEN 사용자가 선택한 제보 유형을 실행하면, THE SpotDetailPage SHALL 해당 기존 폼을 동일 영역 안에서 렌더링한다
4. WHEN 사용자가 폼을 닫거나 제출에 성공하면, THE SpotDetailPage SHALL 선택 상태를 해제하고 통합 영역으로 돌아간다
5. IF 사용자가 로그인하지 않은 상태에서 제보 유형을 실행하면, THEN 기존 로그인 필요 모달 흐름을 유지한다
6. IF 기존 `ContributorList`가 정보 보완 기여자를 표시하고 있다면, THEN 통합 영역 안에서 계속 표시한다

### 요구사항 5: 모바일 지도 필터 점유 면적 축소

**사용자 스토리:** 모바일 사용자로서, 지도 탐색 중 검색/필터가 지도 화면을 과도하게 가리지 않길 원한다.

#### 수용 기준

1. WHEN 사용자가 모바일 화면에서 `/map`을 보면, THE MapPage SHALL 검색/필터 패널을 접고 펼칠 수 있는 버튼을 제공한다
2. WHEN 필터 패널이 접혀 있으면, THE MapPage SHALL 지도 위에 최소한의 필터 열기 버튼만 표시한다
3. WHEN 필터 패널이 펼쳐지면, THE MapPage SHALL 기존 콘텐츠 검색과 카테고리 필터를 제공한다
4. WHEN 사용자가 데스크톱 화면에서 `/map`을 보면, THE MapPage SHALL 기존처럼 검색/필터 바를 즉시 표시한다
5. WHILE 필터 변경으로 데이터가 갱신 중이면, THE MapPage SHALL 기존 로딩 진행 표시를 유지한다
