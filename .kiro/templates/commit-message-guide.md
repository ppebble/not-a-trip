# 커밋 메시지 작성 가이드

## 📋 커밋 메시지 규칙

### 기본 형식

```
<type>: <description>

<body>
```

### Type 종류

- `feat:` - 새로운 기능 추가
- `fix:` - 버그 수정
- `style:` - UI/스타일 변경 (코드 동작에 영향 없음)
- `refactor:` - 코드 리팩토링 (기능 변경 없음)
- `test:` - 테스트 추가/수정
- `docs:` - 문서 변경
- `chore:` - 빌드 프로세스, 도구 설정 등

### Description 규칙

- 50자 이내로 작성
- 첫 글자는 소문자
- 마침표 없음
- 명령형으로 작성 ("추가한다" X, "추가" O)

## 📝 커밋 메시지 예시

### 좋은 예시

```bash
feat: spotpin 컴포넌트 기본 구현

- 마커 아이콘 생성 함수 추가
- 클릭 이벤트 핸들링 구현
- 네이비 테마 적용
```

```bash
style: spotpin 마커 ui/ux 개선

- 마커 크기별 상태 관리 (기본/호버/선택)
- 그라데이션 배경과 향상된 그림자 효과
- 선택된 마커 펄스 애니메이션과 링 효과
- Requirements 1.2, 2.1 대응
```

```bash
fix: next/image 외부 이미지 도메인 설정 추가

- picsum.photos, images.unsplash.com 도메인 추가
- Next.js 15 remotePatterns 설정으로 외부 이미지 최적화 지원
- Requirements 2.2 대응
```

### 나쁜 예시

```bash
# 너무 모호함
fix: 버그 수정

# 너무 길고 구체적이지 않음
feat: SpotPin 컴포넌트를 구현하고 마커를 추가했습니다.

# 타입이 잘못됨
update: 스타일 변경
```

## 🚨 커밋 분할 전략

### 논리적 단위로 분할

1. **기능별 분할**

   ```bash
   feat: spotpin 컴포넌트 기본 구현
   style: spotpin 마커 스타일링 추가
   feat: spotpin 클릭 이벤트 핸들링 구현
   ```

2. **파일 유형별 분할**
   ```bash
   feat: next/image 외부 이미지 도메인 설정 추가
   style: spotpin 마커 ui/ux 개선
   style: 메인 페이지 레이아웃 및 반응형 디자인 개선
   ```

### 커밋 크기 가이드

- **이상적**: 100-300줄
- **최대**: 500줄
- **500줄 초과시**: 여러 커밋으로 분할

## 📖 Body 작성 가이드

### Body 포함 시기

- 변경사항이 복잡한 경우
- Requirements 참조가 필요한 경우
- 여러 파일이 변경된 경우

### Body 작성 형식

```bash
feat: spotpin 컴포넌트 기본 구현

- 마커 아이콘 생성 함수 추가
- 클릭 이벤트 핸들링 구현
- 네이비 테마 적용
- Requirements 1.2, 2.1 대응
```

### Body 작성 팁

- 각 항목은 `-`로 시작
- 구체적인 변경사항 나열
- Requirements 번호 포함
- 72자 이내로 줄바꿈

## 🔗 Requirements 참조

커밋 메시지에 관련 Requirements를 명시:

```bash
feat: 스팟 미리보기 팝업 구현

- 팝업 UI 컴포넌트 추가
- 필수 정보 표시 (이름, 사진, 설명, 주소)
- 상세보기 버튼 구현
- Requirements 2.2, 2.3, 2.4 대응
```

## ⚠️ 주의사항

1. **한글 사용**: 모든 커밋 메시지는 한글로 작성
2. **일관성**: 팀 내 커밋 메시지 스타일 통일
3. **명확성**: 변경사항을 명확하게 설명
4. **추적성**: Requirements 번호로 추적 가능하도록 작성
5. **원자성**: 하나의 커밋은 하나의 논리적 변경사항만 포함
